文件格式

标签定义文件使用JSON格式，并具有下列结构：

 - NBT复合标签/JSON对象 根标签。
     - 布尔型replace：（默认为false）此标签是否完全覆盖较低优先级数据包同命名空间下的同名标签。为false时则为此标签进行补充。
 - NBT列表/JSON数组values：一个游戏资源的列表，代表此标签的内容。
     - 字符串：一项游戏资源，格式应为<命名空间>:<路径>。
     - 字符串：一项游戏资源的标签，格式应为#<命名空间>:<路径>。循环引用会导致标签加载失败。
 - NBT复合标签/JSON对象：带有附加选项的格式。
     - 字符串id：一个游戏资源的ID或其标签ID，格式同上。
     - 布尔型required：（默认为true）字符串id指定的游戏资源是否是必须的。为false时，即使字符串id所述内容不存在，游戏也只会静默忽略，而不会使标签加载失败。
